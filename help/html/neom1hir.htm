Limitations

Because NeoMem reads the entire file into memory, the number of objects and text it can contain is limited by the amount of memory available on your computer, and the memory used by other applications you have running.

Each object can store up to 16MB of text information in the Text View.

You can see how much memory your file is taking up by selecting File / File Properties from the main menu. The dialog will also show how many objects your file contains (see the File Properties Dialog topic).

If your file starts getting too large, you can always split your file into two by saving it with a new name, then deleting objects from each file so that they are of a more manageable size.

For network users: please note that NeoMem files can only be used by one person at a time. Files are NOT locked during use, so be careful if you store NeoMem files on a network, as someone else could overwrite your changes if they use the same file at the same time.

Other Limitations

Other limitations of the current version:

  • Copying and Pasting objects is not implemented

  • Dragging and Dropping objects between files is not implemented

  • Merging files together is not implemented

  • Person names are currently only parsed into first and last names
