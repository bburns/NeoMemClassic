Frequently Asked Questions (FAQ)

If you have a question that is not in this list, you can also check the latest FAQ at www.neomem.org/support.htm.

Q. What's the difference between a folder object and other types of objects?

A. Like all other objects, a folder object can store child objects and display them in the Contents View. Unlike other types of objects though, a folder object always gets its own unique column arrangement in the Contents View. Other objects use a column arrangement that is common to their class. For instance, if you had a Tape object with Song, Band, and Length properties displayed, all Tape objects would get the same column arrangement, and if you changed the arrangement in one tape, all other tapes would have the same arrangement.

The best way to arrange your file is to use folder objects to store your other objects. See the example file for some examples.

Q. Why can't I move items up and down? The Move Up and Move Down buttons are disabled.

A. The Move Up and Move Down buttons are enabled only if the currently selected object's parent has autosort turned off. If you right click on the parent object, you can turn off autosort for the object by unchecking the menu item. Then you can arrange the child objects in any order you wish.
