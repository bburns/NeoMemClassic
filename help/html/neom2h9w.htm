Import and Export

Import

If you have information in text files that you would like to import into NeoMem, you can do so by selecting the object that will be the parent of the new information, then selecting File / Import... from the main menu. Currently only text or rich text files (rtf) are supported.

Export

You can export the contents of the entire file to a Comma Separated Values (csv) text file by selecting File / Export... from the main menu. Or, to export a selected object and its child objects, right click on it in the Data View, and select Export Object (and children)... from the popup menu.

The resulting csv file will include columns with object id's where appropriate, for instance any link properties will have a column showing a human readable version, and another column for the object id's linked to (for multiple objects, they will just be separated by commas).

The last two columns in the file will be the contents of the Text View, one column for the rtf version, and one for the plain text version.

Warning: Importing the resulting csv file may fail if your Text Views have a lot of text in them - for instance, Excel has a limit of 32k characters per cell - beyond that and the text will spill over into following cells. And Access has a limit of 65k characters per row in the text file. Beyond that and it crashes (tested on Excel 2003 and Access 2003).

Also, to import correctly into Excel, you should use File/Open, and NOT Data/Import External Data, because the latter does not correctly interpret the line breaks in the Text View fields. But using File/Open will work properly.

To export data from the Contents View, you can copy the data to the clipboard by clicking in the Contents View to select it as the current view, then selecting Edit / Copy Grid Contents and then pasting the data into a spreadsheet program or word processor. The cell data will be separated by tab characters, which most spreadsheets will be able to parse correctly.

You can also export data from the Properties View as above for the Contents View.
