Views

Contents View

The Contents View shows the objects that the currently selected object contains. You can use this view like a spreadsheet, entering information in the cells and adding new child objects using the new row (marked with a triangle).

Adding Objects

When adding a object on the new row, the class of the object is determined by the Default Class of the current object. For instance, in the example file, if you selected the Authors folder, then entered a new author's name on the new object row, the class of the new object would be Author, because the Authors folder specifies the Author class as its Default Class. See the Change Contents topic for more information.

Selecting Objects

To select an object in the Contents View as the current object, you can double click on its name or icon, hit Ctrl+Enter with the object's name highlighted, or right click on its name and select Open Object from the popup menu.

Editing Values

Double clicking on any property value (except for the Name property) will bring up the Edit Dialog that is appropriate for that property type - hitting F4 while on a cell will have the same effect. Hitting F2 while on a cell will let you edit its contents in place, or you can just start typing over the existing value.

If you right click on a property value, you will get a popup menu that will give you access to the clipboard, and depending on the property type, will also display other commands. For instance, a Link property will display menu items letting you jump to the objects that are being linked to.

Column Headers

You can also right click on the column headers for another popup menu, which will let you sort the contents by that column, edit the property associated with the column, and add and remove columns.

To change the order of columns, you can drag and drop them to their new location.

Note: All Folder objects get their own unique column arrangement, but all other objects share an arrangement that is stored with their class (for instance, all Book objects will always have the same column arrangement).
