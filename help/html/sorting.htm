Sorting Objects, Autosort, Moving Items Up and Down

Normally all objects in NeoMem are automatically sorted alphabetically. For certain classes of objects, this may not be appropriate. For instance, if you're storing a list of songs on a CD, the order is not necessarily alphabetic. In this case, you can modify the CD class so that its child items will not be sorted. You can do this in the Class Wizard when you are setting up a class, or modifying an existing class. You can also turn autosort on or off for a single object by right clicking on the object and selecting Sort Subobjects Automatically to it on and off (you can also click on the object and select Object / Sort Subobjects Automatically from the main menu).

For objects which have autosort turned off, you can change the order of the child objects by clicking on a child object and then clicking on the Move Object Up or Move Object Down buttons (or selecting Object / Move Up or Move Down from the main menu). This will move the object up or down relative to its siblings.

In the Contents View, you can temporarily sort the items by one of the columns by clicking on that column header. Clicking on it again will reverse the sort. You can also right click on a column header and select Sort Ascending or Sort Descending (or click on a cell in the column and select View / Sort Column Ascending or Sort Column Descending from the main menu).

Note: If you are sorting by a date column and any of the dates are stored as strings, they will always be sorted at the end - see Properties and Property Types for more information.

You can also add a non-alphanumeric character (e.g. "_" or "@") before the name of an item that you want to appear first in a sort. This may be preferable to turning autosort off in some cases.
